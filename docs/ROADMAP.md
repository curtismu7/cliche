# ClipShot Roadmap

Research survey of open-source clipboard managers and screen capture tools,
with prioritized next-phase recommendations. Compiled 2026-07-04.

## Survey: What Users Love in Comparable Apps

**Maccy** ([github.com/p0deje/Maccy](https://github.com/p0deje/Maccy)) — The gold standard for keyboard-first clipboard management on macOS (~8k+ stars). Standout features: instant fuzzy search over thousands of entries ("hit hotkey, type, Enter"), paste directly into the frontmost app (Option+Enter), paste without formatting (Option+Shift+Enter), number-key shortcuts for the first nine items, pinning, and a carefully designed privacy story — it ignores `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType` plus a configurable list of app-specific pasteboard types (1Password, KeePass, etc.), so passwords never enter history. Accessibility permission is requested lazily — only the first time you actually paste, and the app works copy-only without it. Everything is local; no network.

**CopyQ** ([github.com/hluk/CopyQ](https://github.com/hluk/CopyQ)) — The power-user option (cross-platform, Qt). Standout features: multiple tabs for categorizing clips, pinned items that hold their row position, in-place editing of clipboard entries, and a JavaScript-like scripting engine plus CLI (`copyq eval`) for automation. Its lesson for ClipShot: editing and organizing clips is valued, but the Qt/non-native feel is the most common complaint on macOS — a native SwiftUI app that borrows the tab/pin/edit ideas wins.

**Clipy** ([github.com/Clipy/Clipy](https://github.com/Clipy/Clipy)) — ClipMenu's Swift successor (~8.7k stars). Its enduring differentiator is **snippets**: user-defined reusable text organized in folders, summoned from the same menu as history via its own hotkey. History and boilerplate in one tool is the feature users cite most.

**PasteBar** ([github.com/PasteBar/PasteBarApp](https://github.com/PasteBar/PasteBarApp)) — Newer Tauri/Rust entrant, well received on HN. Standout features: unlimited history across text/images/files/links, organization into collections/tabs/boards, custom clips with template form fields, a Quick Paste window where typing immediately starts searching (zero-friction search activation), and lock-screen/passcode protection. Local-only storage is heavily marketed — privacy is a selling point in this category.

**Shottr** ([shottr.cc](https://shottr.cc)) — Not OSS, but the most-praised indie screenshot tool on HN/Reddit; tiny (2–3 MB), native, Apple Silicon-optimized. Standout features: near-instant capture (~17 ms), OCR of any screen region to clipboard (also reads QR codes), scrolling capture of long pages/chats, annotation (arrows, shapes, text, counters), pixelation/redaction that actually removes data, pin-screenshot-as-floating-window, and pixel measurement/color tools. Its core lesson: speed + OCR + lightweight annotation beats feature bloat.

**Flameshot** ([github.com/flameshot-org/flameshot](https://github.com/flameshot-org/flameshot)) — Beloved Linux-first OSS tool. Standout features: annotation happens *in the capture overlay itself* (draw arrows/boxes/blur before the screenshot is even saved — one fewer step than capture-then-edit), incrementing counter badges (great for step-by-step docs), pin-to-desktop floating screenshots, and full CLI control. The in-selection editing flow is its signature UX.

**Kap** ([github.com/wulkano/Kap](https://github.com/wulkano/Kap)) — OSS menu bar screen recorder (Electron). Standout features: region/window recording exported as GIF/MP4/WebM/APNG with trimming and click highlighting, plus a plugin system for uploads. Loved by developers for README GIFs. Lesson: even basic recording-to-GIF from the menu bar fills a real gap.

**CleanShot X** ([cleanshot.com/features](https://cleanshot.com/features)) — The commercial benchmark ("7 apps in 1", 50+ features). The features users cite most in reviews: the **Quick Access Overlay** (floating thumbnail after capture — drag it into Slack, click to annotate, dismiss to save), scrolling capture, the annotation editor, capture history, OCR, and one-click cloud share links. The overlay-after-capture workflow is the single most copied UX pattern in this space.

## 1. High Value / Low Effort (quick wins)

- **Launch at login** — table stakes for a menu bar utility (all surveyed apps). *Impl:* `SMAppService.mainApp.register()` / `.unregister()` (macOS 13+), a single toggle in settings; no permissions.
- **Password-manager ignore list** — never record concealed/transient clipboard content (Maccy). *Impl:* ClipShot already skips `ConcealedType`/`TransientType`; add `AutoGeneratedType` plus a user-editable list of ignored pasteboard types and app bundle IDs (`NSWorkspace.shared.frontmostApplication`). Small code, big trust win.
- **Fuzzy search of history** — type-to-filter the history popup (Maccy, PasteBar). *Impl:* pure Swift subsequence/scored matcher, `TextField` auto-focused when the panel opens. No permissions.
- **Full keyboard navigation** — arrow keys, Return to copy, ⌘1–⌘9 for top items, Delete to remove, P to pin (Maccy). *Impl:* SwiftUI `.onKeyPress` (macOS 14+) or a local `NSEvent` key-down monitor on the panel.
- **Paste as plain text** — strip formatting on demand (Maccy Option+Shift+Enter). *Impl:* modifier-key variant of the existing copy action.
- **Capture history** — a browsable grid of past screenshots with copy/reveal/delete (CleanShot X, Shottr). *Impl:* point `screencapture` output at `~/Library/Application Support/ClipShot/Captures/`, index in the existing JSON store, `LazyVGrid` thumbnails.
- **Pin screenshot as floating window** — keep a capture always-on-top for reference (Shottr, Flameshot). *Impl:* borderless `NSPanel` with `.level = .floating`, drag-anywhere + Esc to close.

## 2. High Value / Higher Effort

- **OCR / copy text from screen (Vision)** — dedicated hotkey: select region → recognized text lands on the clipboard (Shottr's most-loved feature). *Impl:* region capture → `VNRecognizeTextRequest` (`.accurate`), join observations, write to pasteboard and history. All on-device; no new permission. Lowest-effort item in this group — start here.
- **Paste directly into frontmost app** — select a history item and it pastes where you were typing (Maccy). *Impl:* write to pasteboard, close panel, synthesize ⌘V via `CGEvent` posted to `.cgAnnotatedSessionEventTap`. **Requires Accessibility permission** — request lazily via `AXIsProcessTrustedWithOptions` only on first paste; degrade to copy-only if denied. Panel must be a non-activating `NSPanel` so focus stays in the target app.
- **Post-capture annotation editor** — arrows, rectangles, text, highlight, pixelate/redact, counter badges (Shottr, Flameshot, CleanShot X). *Impl:* SwiftUI `Canvas` over the captured `NSImage`, value-type annotation array (undo via snapshots), flatten with `ImageRenderer`; pixelation via `CIPixellate`. Scope tightly: arrows + rect + text + blur + counters covers ~90% of usage.
- **Quick Access Overlay after capture** — floating thumbnail in the screen corner: drag into another app, click to annotate, auto-dismiss to save (CleanShot X's signature UX). *Impl:* small non-activating `NSPanel` at `.statusBar` level, `onDrag` for drag-out, timer for auto-dismiss.
- **Snippets** — user-defined reusable text with placeholder variables (`%DATE%`, `%CLIPBOARD%`) alongside history (Clipy, PasteBar). *Impl:* separate panel section + existing JSON persistence; substitution at copy time.
- **Migrate capture to ScreenCaptureKit** — replace the `screencapture` CLI with `SCScreenshotManager`. Unlocks in-process capture (needed for the overlay/annotation flow to feel instant), window filtering, cursor control. Uses the same Screen Recording permission; keep the CLI path as fallback. Enabler for annotation, overlay, and recording — sequence it before or alongside them.

## 3. Nice-to-Have / Later

- **Scrolling capture** — stitch a long page/chat into one tall image (Shottr, CleanShot X). Genuinely hard to get robust (fixed headers, elastic scrolling); high wow-factor, high maintenance. Defer until the ScreenCaptureKit migration is done.
- **Screen recording → GIF/MP4** — region/window recording from the menu bar (Kap, CleanShot X). *Impl:* `SCStream` → `AVAssetWriter` for MP4; GIF via `CGImageDestination`. Big scope — its own milestone.
- **Tabs/boards for clips** — organize saved clips into named groups (CopyQ, PasteBar). Only worth it once users accumulate pinned items.
- **Edit clip in place** — modify a history item's text before reuse (CopyQ). Small, but low demand until search/paste exist.
- **Share/upload** — `NSSharingServicePicker` on captures (AirDrop, Mail, Messages) as a free step before considering cloud links.
- **Color picker & pixel measurement** — `NSColorSampler` gives a native magnifier loupe for free. Fun, cheap, off the critical path.
- **URL scheme / scripting** — `clipshot://capture?mode=region` for Raycast/Shortcuts users; the cheap 80% version of CopyQ-style scripting.

**Suggested sequencing:** ship all of group 1 as one release (ignore-list first), then OCR → paste-directly → ScreenCaptureKit → annotation + overlay as the following two releases. That ordering front-loads the two features (fuzzy search + OCR) that most strongly drive word-of-mouth for Maccy and Shottr respectively.
